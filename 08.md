# Load Balancing
1. Find out the exposed IP and Port:
    ```
    $ kubectl describe services
    ```
    Response:
    ```
    Name:              kubernetes
    Namespace:         default
    Labels:            component=apiserver
                      provider=kubernetes
    Annotations:       <none>
    Selector:          <none>
    Type:              ClusterIP
    IP:                10.96.0.1
    Port:              https  443/TCP
    TargetPort:        8443/TCP
    Endpoints:         192.168.99.107:8443
    Session Affinity:  None
    Events:            <none>
    ```
2. Create a new service:
    ```
    $ kubectl expose deployment/kubernetes-bootcamp --type="NodePort" --port 8080
    ```
    Response:
    ```
    service/kubernetes-bootcamp exposed
    ```
3. See service detail:
    ```
    $ kubectl describe services/kubernetes-bootcamp
    ```
    Response:
    ```
    Name:                     kubernetes-bootcamp
    Namespace:                default
    Labels:                   app=kubernetes-bootcamp
    Annotations:              <none>
    Selector:                 app=kubernetes-bootcamp
    Type:                     NodePort
    IP:                       10.110.229.147
    Port:                     <unset>  8080/TCP
    TargetPort:               8080/TCP
    NodePort:                 <unset>  30086/TCP
    Endpoints:                172.17.0.5:8080,172.17.0.6:8080,172.17.0.8:8080 + 1 more...
    Session Affinity:         None
    External Traffic Policy:  Cluster
    Events:                   <none>
    ```
4. Create env variable:
    ```
    $ export NODE_PORT=$(kubectl get services/kubernetes-bootcamp -o go-template='{{(index .spec.ports 0).nodePort}}')
    ```
5. Test exposed IP and port:
    ```
    $ curl $(minikube ip):$NODE_PORT
    ```
    We hit a different Pod every request.
